# Code of Conduct

Quant Evo Next-Gen is an engineering-heavy project, but the standard for collaboration is still simple:

- be respectful
- be specific
- assume good intent, but do not hide real technical concerns
- prefer evidence, repro steps, and concrete operator impact over personal judgment

## Expected behavior

- Give clear, honest feedback.
- Critique ideas, code, docs, and operations, not people.
- Share enough context for others to reproduce and understand an issue.
- Respect security boundaries, private reports, and redacted data requirements.
- Keep discussions useful for operators, contributors, and maintainers.

## Unacceptable behavior

- Harassment, abuse, threats, or personal attacks
- Publishing private credentials, tokens, or operator data
- Dismissing legitimate safety, risk, or deploy concerns without review
- Spamming, derailing, or repeatedly reopening settled threads without new evidence

## Scope

This applies to repository discussions, issues, pull requests, and other project communication channels.

## Reporting

If you run into unacceptable behavior, contact the maintainer privately.

If the report also involves security, secrets, broker access, or live-trading risk, follow [SECURITY.md](SECURITY.md) instead of opening a public issue.
